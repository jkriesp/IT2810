Css grid or Flexbox

I decided to use Flexbox layout for the website. The design of the website is not complex and using flexbox I find it easier to align content within

containers. My take on CSS grid is that it’s more relevant to use if you’re implementing a two-dimensional website – A left aligned menu bellow a header for example.

HTML Canvas og SVG

In the above illustration I’ve tried to implement the same “abstract” smiley using SVG and HTML Canvas. There were several hurdles to overcome, one that I didn’t manage to solve. First the major difference between SVG and Canvas is that SVG uses HTML elements. That in turn makes it a lot more flexible to use CSS to design and animate. Canvas is just pixels, which in turn makes it harder to target specific shapes you want to animate or design. Animating the canvas shapes has to be done programmatically, which I didn’t manage to solve in time.

The SVG picture is animated using basic CSS animation.

Another hurdle I came across implementing the artwork using HTML canvas was in regard to colouring the elements. I managed to head of the smiley just fine, but when trying to colour each eye the entire face was filled with that colour. That leads me to believe that the eyes are not closed circles, and the fill-method does just that, fills the entire shape.

jQuery

I’ve used jQuery to manage the “Documentation” toggle-button. I realize jQuery is a powerful library, but in this task, I didn’t find more use of it. I thought there might be use of it in regard to the canvas drawing, and found some alternatives, but I felt like the teaching goals wouldn’t be as clear if I didn’t try to implement the HTML canvas picture using regular HTML5.

I did in turn find it easier to use jQuery to manipulate the DOM when my goal was to add a click listener to the toggle-button. Which I learned is the main goal of jQuery, easier and quicker DOM manipulation.
